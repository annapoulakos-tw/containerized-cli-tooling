# Skill: Minimal Code

## Definition

Solve the problem with the smallest clear change.

## Rules

- Prefer existing patterns.
- Prefer existing dependencies.
- Prefer inline idiomatic code for simple local operations.
- Do not create helper functions for one-line, one-use transformations.
- Do not introduce classes, factories, registries, frameworks, or extension points unless required by the spec or existing repository conventions.
- Avoid speculative abstractions.
- Avoid future-proofing.
- Avoid generic frameworks.
- Remove unnecessary code when discovered.
- Stop when acceptance criteria are satisfied.
